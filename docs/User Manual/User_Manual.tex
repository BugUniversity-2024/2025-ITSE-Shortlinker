% ==================== Chapter 1: Introduction ====================
\section{Introduction}

\subsection{About TinyBridge}

TinyBridge is a lightweight, efficient URL shortening service designed for campus organizations, small  enterprises and individual content creators. The platform transforms long, unwieldy URLs into concise, memorable short links while providing comprehensive analytics and customization features.

\subsection{Key Features}

TinyBridge offers the following core functionalities:

\begin{itemize}
    \item \textbf{URL Shortening}: Convert long URLs into short, shareable links using a 6-character alphanumeric code
    \item \textbf{Custom Short Codes}: Create personalized short codes for better brand recognition
    \item \textbf{Click Analytics}: Track click counts, geographic distribution, device types, and traffic sources
    \item \textbf{QR Code Generation}: Automatically generate QR codes for each short link
    \item \textbf{API Access}: Programmatic access via RESTful API with key-based authentication
\end{itemize}

% ==================== Chapter 2: Installation Guide ====================
\section{Installation Guide}

This chapter provides step-by-step instructions for installing and configuring TinyBridge on your local machine.

\subsection{Prerequisites}

\subsubsection{Installing Node.js}

Download and install Node.js from the official website: \url{https://nodejs.org/}

Verify the installation by running:
\begin{lstlisting}[language=bash]
node --version
\end{lstlisting}

\subsubsection{Installing Yarn}

Install Yarn via corepack (bundled with Node.js 16.10+):
\begin{lstlisting}[language=bash]
corepack enable
yarn --version
\end{lstlisting}

\subsection{Backend Installation}

\subsubsection{Step 1: Clone the Repository}

\begin{lstlisting}[language=bash]
git clone https://github.com/BugUniversity-2024/2025-ITSE-Shortlinker.git
cd 2025-ITSE-Shortlinker/Backend
\end{lstlisting}

\subsubsection{Step 2: Install Dependencies}

\begin{lstlisting}[language=bash]
yarn install
\end{lstlisting}

\subsubsection{Step 3: Configure Environment Variables}

Create a \texttt{.env} file in the Backend directory:
\begin{lstlisting}
NODE_ENV=development
PORT=8000
DATABASE_URL="file:./dev.db"
JWT_SECRET=your-secret-key-here
JWT_EXPIRES_IN=24h
CORS_ORIGINS=http://localhost:5173
\end{lstlisting}

\textbf{Important}: Replace \texttt{your-secret-key-here} with a secure random string for production use.

\subsubsection{Step 4: Initialize the Database}

\begin{lstlisting}[language=bash]
yarn db:push
\end{lstlisting}

This command creates the SQLite database and initializes all required tables.

\subsubsection{Step 5: Start the Backend Server}

For development:
\begin{lstlisting}[language=bash]
yarn dev
\end{lstlisting}

The backend server will start at \texttt{http://localhost:8000}.

\subsection{Frontend Installation}

\subsubsection{Step 1: Navigate to Frontend Directory}

\begin{lstlisting}[language=bash]
cd ../Frontend
\end{lstlisting}

\subsubsection{Step 2: Install Dependencies}

\begin{lstlisting}[language=bash]
yarn install
\end{lstlisting}

\subsubsection{Step 3: Start the Development Server}

\begin{lstlisting}[language=bash]
yarn dev
\end{lstlisting}

The frontend application will be available at \texttt{http://localhost:5173}.

\subsection{Verifying Installation}

After starting both servers, open your browser and navigate to \texttt{http://localhost:5173}. You should see the TinyBridge login page as shown in Figure~\ref{fig:login}.

% ==================== Chapter 3: Quick Start Guide ====================
\section{Quick Start Guide}

\subsection{Step 1: Create an Account}

\begin{enumerate}
    \item Open TinyBridge in  \texttt{http://localhost:5173}
    \item Click ``Register now'' on the login page
    \item Fill in the registration form:
    \begin{itemize}
        \item Username (3-20 characters)
        \item Email address
        \item Password (minimum 8 characters)
    \end{itemize}
    \item Click ``Create Account''
\end{enumerate}

\subsection{Step 2: Log In}

\begin{enumerate}
    \item Enter your email address and password. Click ``Sign In''. 
\end{enumerate}

\subsection{Step 3: Create Your First Short Link}

\begin{enumerate}
    \item Click ``Create Link''. Enter the original URL you want to shorten
    \item (Optional) Customize the short code, add a description or set an expiration time
    \item Click ``Generate Short Link''.Copy the generated short link to share
\end{enumerate}

\subsection{Step 4: Test the Short Link}

Open a new browser tab and paste your short link. You should be redirected to the original URL.

\subsubsection{Creating an API Key}
\begin{enumerate}
    \item Click ``Create New Key''
    \item Enter a name for the key (e.g., ``Production'', ``Testing'')
    \item Select permissions: \texttt{links:read}, \texttt{links:write}, \texttt{links:delete}, \texttt{analytics:read}
    \item Set rate limits and expiration (optional)
    \item Click ``Create''
    \item \textbf{Important}: Copy the key immediately as it won't be shown again
\end{enumerate}

\subsubsection{Using the API}
Include the API key in the Authorization header:
\begin{lstlisting}
curl -X POST https://api.tinybridge.link/v1/links \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"url": "https://example.com"}'
\end{lstlisting}

% ==================== Chapter 5: Frequently Asked Questions ====================
\section{Frequently Asked Questions (FAQs)}

\noindent\textbf{\underline{Q1: What is TinyBridge?}}\\
A1: TinyBridge is a lightweight URL shortening service that transforms long URLs into short, memorable links with analytics and API access.

\noindent\textbf{\underline{Q2: Is TinyBridge free to use?}}\\
A2: Yes, TinyBridge is completely free. All features including link creation, analytics, and API access are available at no cost.

\noindent\textbf{\underline{Q3: How do I create a short link?}}\\
A3: Navigate to ``Create Link'', enter your URL, optionally customize the short code, and click ``Generate Short Link''.

\noindent\textbf{\underline{Q4: Can I customize my short links?}}\\
A4: Yes, you can create custom short codes (4-20 characters, alphanumeric with hyphens and underscores).

\noindent\textbf{\underline{Q5: How do I track link performance?}}\\
A5: Visit the ``Analytics'' page to view click counts, geographic distribution, device types, and traffic sources.

\noindent\textbf{\underline{Q6: Is my data secure?}}\\
A6: Yes. All passwords are encrypted using industry-standard algorithms, and user data is only accessible to the account owner.

\noindent\textbf{\underline{Q7: How do I use the API?}}\\
A7: Generate an API key from ``API Keys'' page, then use the header: \texttt{Authorization: Bearer YOUR\_API\_KEY}.

\noindent\textbf{\underline{Q8: What browsers are supported?}}\\
A8: Chrome, Firefox, Safari, and Edge. We recommend using the latest version for optimal performance.

% ==================== Chapter 6: Troubleshooting ====================
\section{Troubleshooting}

\noindent\textbf{\underline{1. Unable to register or log in}}\\
Verify that email is valid and not already registered, password meets requirements (8+ chars), and network is stable. Try clearing browser cache if issues persist.

\noindent\textbf{\underline{2. Short link not redirecting}}\\
Check that the link has not been disabled/deleted, has not expired, and the original URL is still accessible.

\noindent\textbf{\underline{3. Analytics data not showing}}\\
Analytics may take a few minutes to update. Ensure you're viewing the correct link and that it has received clicks.

\noindent\textbf{\underline{4. API requests returning errors}}\\
Verify your API key is valid, has appropriate permissions, and request format matches the documentation.

\noindent\textbf{\underline{5. Page loading slowly}}\\
Check internet connection, clear browser cache, try a different browser, and ensure JavaScript is enabled.
